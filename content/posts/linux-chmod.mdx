서버에 파일을 올렸는데 "Permission denied"가 뜨거나, 반대로 민감한 설정 파일이 누구나 읽을 수 있도록 열려 있다면 권한 설정이 잘못된 것이다. Linux의 파일 권한 모델을 이해하면 이런 문제를 정확히 진단하고 수정할 수 있다.

## 권한 확인: ls -l

```bash
$ ls -l
-rwxr-xr--  2  ubuntu  www-data  4096  Apr  6 10:00  deploy.sh
drwxr-x---  3  ubuntu  www-data  4096  Apr  6 09:00  config/
```

첫 번째 열의 10자리가 권한 정보다.

```
- r w x r - x r - -
│ │─────│ │─────│ │─────│
│  owner   group   others
│
└── 파일 종류: - (일반 파일), d (디렉터리), l (심볼릭 링크)
```

---

## 세 가지 권한

| 기호 | 숫자 | 이름 | 파일에서의 의미 | 디렉터리에서의 의미 |
|------|------|------|--------------|-----------------|
| `r` | 4 | Read | 파일 내용 읽기 | 디렉터리 목록 조회 (`ls`) |
| `w` | 2 | Write | 파일 내용 수정·삭제 | 파일 생성·삭제 |
| `x` | 1 | Execute | 파일 실행 | 디렉터리 진입 (`cd`) |
| `-` | 0 | None | 해당 권한 없음 | 해당 권한 없음 |

> 디렉터리에서 `x`(실행) 권한이 없으면 `cd`로 진입할 수 없다. `r`만 있으면 `ls`로 목록은 볼 수 있지만 내부 파일에 접근하지 못한다.

---

## 세 가지 대상

| 대상 | 기호 | 설명 |
|------|------|------|
| **Owner** | `u` | 파일 소유자 |
| **Group** | `g` | 파일의 그룹 소속 사용자들 |
| **Others** | `o` | 소유자도 그룹도 아닌 나머지 사용자 |
| **All** | `a` | 위 세 대상 모두 |

## chmod 사용법

### 숫자(8진수) 표기법

세 대상의 권한을 각각 숫자로 더해서 표현한다.

```
r=4, w=2, x=1, 없음=0

rwx = 4+2+1 = 7
rw- = 4+2+0 = 6
r-x = 4+0+1 = 5
r-- = 4+0+0 = 4
-wx = 0+2+1 = 3
-w- = 0+2+0 = 2
--x = 0+0+1 = 1
--- = 0+0+0 = 0
```

```bash
chmod 755 deploy.sh
# u=rwx(7), g=r-x(5), o=r-x(5)
# → 소유자: 읽기·쓰기·실행 / 그룹·기타: 읽기·실행

chmod 644 config.yml
# u=rw-(6), g=r--(4), o=r--(4)
# → 소유자: 읽기·쓰기 / 그룹·기타: 읽기만

chmod 600 .env
# u=rw-(6), g=---(0), o=---(0)
# → 소유자만 읽기·쓰기 (민감 파일에 사용)

chmod 777 /tmp/shared
# 모두 읽기·쓰기·실행 (보안상 위험, 최소화)
```

### 기호 표기법

현재 권한에서 특정 권한만 추가(`+`)하거나 제거(`-`), 또는 정확히 설정(`=`)한다.

```bash
# 실행 권한 추가
chmod +x script.sh          # 모든 대상에 실행 권한 추가
chmod u+x script.sh         # 소유자에게만 실행 권한 추가

# 권한 제거
chmod o-r secret.txt        # 기타 사용자의 읽기 권한 제거
chmod go-w important.conf   # 그룹과 기타 사용자의 쓰기 권한 제거

# 권한 정확히 설정 (나머지는 제거)
chmod u=rwx,g=rx,o=r deploy.sh

# 여러 대상 동시 설정
chmod a+r public.html        # 모든 사용자에 읽기 추가
chmod ug+x,o-x run.sh       # 소유자·그룹에 실행 추가, 기타는 실행 제거
```

## 자주 쓰는 권한 조합

| 권한 | 숫자 | 용도 |
|------|------|------|
| `rwxr-xr-x` | 755 | 실행 파일, 공개 디렉터리 (표준) |
| `rw-r--r--` | 644 | 일반 파일, 설정 파일 (표준) |
| `rw-------` | 600 | SSH 개인 키, `.env`, 비밀 파일 |
| `rwx------` | 700 | 개인 스크립트, 비공개 디렉터리 |
| `rwxrwxr-x` | 775 | 팀 공유 디렉터리 |
| `rw-rw-r--` | 664 | 팀 공유 파일 |
| `rwxrwxrwx` | 777 | 누구나 접근 (보안 위험, 지양) |

```bash
# SSH 키 권한 — 틀리면 SSH 접속 거부됨
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519        # 개인 키
chmod 644 ~/.ssh/id_ed25519.pub    # 공개 키
chmod 600 ~/.ssh/authorized_keys
chmod 644 ~/.ssh/known_hosts
```

## 재귀 적용: -R 옵션

디렉터리와 그 안의 모든 파일·하위 디렉터리에 일괄 적용한다.

```bash
chmod -R 755 /var/www/html

# 주의: -R로 파일과 디렉터리에 같은 권한을 주면 안 될 때
# 파일은 644, 디렉터리는 755로 각각 설정하려면 find를 조합한다
find /var/www -type f -exec chmod 644 {} \;   # 파일만
find /var/www -type d -exec chmod 755 {} \;   # 디렉터리만
```

## 특수 권한

일반 rwx 외에 세 가지 특수 권한이 존재한다.

### setuid (4xxx)

**파일 실행 시 소유자의 권한으로 실행**되게 한다. 실행 비트 자리에 `s`로 표시된다.

```bash
chmod 4755 /usr/bin/passwd
# 또는
chmod u+s /usr/bin/passwd

# 결과
-rwsr-xr-x  root  root  /usr/bin/passwd
    ↑ s = setuid
```

`/usr/bin/passwd`는 일반 사용자가 실행해도 root 권한으로 `/etc/shadow`를 수정할 수 있다. 이것이 setuid 덕분이다.

> 보안상 setuid가 설정된 파일은 공격 대상이 되기 쉽다. 불필요한 파일에 setuid를 주지 않는다.

### setgid (2xxx)

**파일 실행 시 그룹 권한으로 실행**된다. 디렉터리에 적용하면 해당 디렉터리 안에 생성된 파일이 **부모 디렉터리의 그룹을 자동으로 상속**한다.

```bash
chmod 2775 /shared/project
# 또는
chmod g+s /shared/project

# 결과
drwxrwsr-x  ubuntu  devteam  /shared/project
       ↑ s = setgid
```

팀 공유 디렉터리에서 누가 파일을 만들어도 항상 같은 그룹 소속이 되게 할 때 유용하다.

### sticky bit (1xxx)

**디렉터리에 설정하면, 파일 삭제를 소유자만 할 수 있다.** 기타 사용자가 쓰기 권한이 있어도 남의 파일은 지우지 못한다.

```bash
chmod 1777 /tmp
# 또는
chmod +t /tmp

# 결과
drwxrwxrwt  root  root  /tmp
          ↑ t = sticky bit
```

`/tmp`는 모두가 파일을 만들 수 있지만, 자신의 파일만 삭제할 수 있다.

### 특수 권한 숫자 표기

```bash
chmod 4755 file    # setuid + 755
chmod 2755 dir     # setgid + 755
chmod 1777 /tmp    # sticky bit + 777
chmod 6755 file    # setuid + setgid + 755
```

## umask — 기본 권한 설정

새 파일/디렉터리 생성 시 적용되는 **기본 권한 마스크**다.

```bash
umask         # 현재 umask 확인 (보통 0022)
umask 0027    # umask 변경
```

```
파일 기본 최대 권한: 666 (rw-rw-rw-)
디렉터리 기본 최대 권한: 777 (rwxrwxrwx)

umask = 022 일 때:
  파일:      666 - 022 = 644 (rw-r--r--)
  디렉터리:  777 - 022 = 755 (rwxr-xr-x)

umask = 027 일 때:
  파일:      666 - 027 = 640 (rw-r-----)
  디렉터리:  777 - 027 = 750 (rwxr-x---)
```

## 소유자·그룹 변경: chown / chgrp

권한 설정과 함께 자주 쓰이는 명령어다.

```bash
# 소유자 변경
chown ubuntu deploy.sh
chown ubuntu:www-data deploy.sh   # 소유자:그룹 동시 변경

# 그룹만 변경
chgrp www-data /var/www/html

# 재귀 적용
chown -R ubuntu:www-data /var/www/html
```

## 권한 문제 디버깅 패턴

```bash
# 현재 사용자와 그룹 확인
whoami
id

# 파일 권한 확인
ls -la /path/to/file

# 디렉터리 전체 경로의 권한 확인 (상위 디렉터리 권한 문제 찾기)
namei -l /var/www/html/index.html
# 결과 예:
# f: /var/www/html/index.html
# drwxr-xr-x root  root  /
# drwxr-xr-x root  root  var
# drwxr-xr-x root  root  www
# drwxr-xr-x root  www-data html
# -rw-r--r-- ubuntu www-data index.html

# 특정 사용자로 명령 실행 테스트
sudo -u www-data cat /var/www/html/config.php
```

---

Linux 파일 권한의 핵심은 **최소 권한 원칙(Principle of Least Privilege)**이다. 필요한 대상에게 필요한 권한만 부여하고, 특히 민감한 파일(`.env`, 개인 키, 설정 파일)은 소유자 외 접근을 차단하는 것이 기본이다.
