2026년 3월 24일, Anthropic이 Claude에 **Computer Use** 기능을 추가했다. Claude가 사람처럼 마우스를 움직이고, 버튼을 클릭하고, 앱을 열고, 브라우저를 탐색해 작업을 완수한다. 동시에 Claude Code에는 **Auto Mode**와 **브라우저 통합**이 추가됐다.

## 무엇이 달라졌는가

지금까지 AI 어시스턴트는 텍스트를 생성하거나, 미리 정의된 API 커넥터를 통해서만 외부 서비스와 연동할 수 있었다. Computer Use는 이 경계를 허문다.

Claude는 이제 커넥터나 API가 없는 서비스도 사람이 쓰듯이 직접 조작할 수 있다. 화면을 인식하고, 클릭하고, 타이핑하고, 스크롤한다.

```
기존: Claude → API 커넥터 → 서비스 (커넥터가 없으면 불가)
변경: Claude → 직접 화면 인식/클릭 → 모든 앱/서비스
```

## Cowork: 데스크톱 제어

Claude Cowork에 macOS 데스크톱 직접 제어 기능이 추가됐다.

작동 방식은 계층적이다. Claude는 먼저 연결된 API 커넥터(Google Workspace, Slack 등)를 우선 사용한다. 커넥터가 없는 서비스라면 실제 마우스와 키보드로 화면을 조작하는 폴백(fallback)으로 전환한다.

**Dispatch와의 연동**이 인상적이다. iPhone의 Dispatch 앱으로 Claude에게 작업을 지시하고 자리를 비우면, 맥에 돌아왔을 때 작업이 완료되어 있다. "이 스프레드시트 보고서를 PDF로 변환해서 팀 드라이브에 올려줘"라고 하고 나가면 된다.

## Claude Code: Auto Mode

개발자 도구인 Claude Code에 **Auto Mode**가 추가됐다.

기존에는 `claude` 명령어 실행 시 모든 도구 호출에 사용자 승인이 필요했다(또는 `--dangerously-skip-permissions`로 전부 건너뜀). Auto Mode는 그 사이의 중간 지점이다.

각 도구 호출 전에 **분류기(classifier)**가 해당 작업을 평가한다.

```
도구 호출 요청
    ↓
분류기 검토
    ↓
안전 → 자동 실행
위험 → 차단 + Claude가 다른 방법 탐색
```

위험 행동으로 분류되는 것들:
- 대량 파일 삭제
- 민감한 데이터 유출 가능성이 있는 작업
- 악성 코드 실행 패턴

```bash
# Auto Mode 활성화
claude --enable-auto-mode

# 실행 중 Shift+Tab으로 모드 전환 가능
# default → auto → dangerously-skip-permissions
```

현재는 Team 플랜 리서치 프리뷰로 제공되고 있으며, Enterprise와 API 플랜으로 순차 확대 예정이다.

## Claude Code: 브라우저 통합

Claude Code가 Chrome 브라우저에 직접 연결된다. 별도 설정 없이 터미널에서 작업하는 동안 Claude가 브라우저를 함께 제어할 수 있다.

가능해지는 것들:

- 버튼 클릭, 폼 입력, 링크 탐색
- 브라우저 콘솔 로그 읽기 (디버깅)
- 여러 탭 걸친 워크플로우
- **기존 로그인 상태 공유** — 새로 인증할 필요 없음

브라우저 작업은 실제 Chrome 창에서 실시간으로 진행된다. Claude가 무엇을 하는지 클릭, 키입력, 페이지 이동이 눈앞에서 보인다.

개발 워크플로우에서 실용적인 시나리오:

```
1. GitHub 이슈 읽기 (브라우저)
2. 코드 수정 (터미널)
3. 테스트 실행 (터미널)
4. PR 생성 및 제출 (브라우저)
5. 팀 Slack에 알림 (브라우저/커넥터)
```

터미널에서 명령 한 번으로 이 흐름이 자동으로 진행된다.

## 안전 장치

Anthropic은 Computer Use를 "safeguards가 내장된 연구 프리뷰"로 출시했다.

주요 안전 원칙:
- **권한 요청** — 새 앱 접근 전에 반드시 사용자 승인을 구한다
- **Auto Mode 분류기** — 위험 행동 패턴을 사전 차단
- **민감 정보 주의** — Anthropic은 현재 민감한 정보를 다루는 작업에는 사용을 자제할 것을 권고
- **가시성** — 브라우저 작업은 실시간으로 화면에 표시

아직 연구 프리뷰 단계다. 모든 시나리오에서 검증된 것이 아니며, 프로덕션 환경의 민감한 작업에 바로 투입하기에는 이르다.

## 현재 가용성

| 기능 | 대상 | 플랫폼 |
|------|------|--------|
| Cowork 데스크톱 제어 | Claude Pro, Max | macOS |
| Claude Code Auto Mode | Team 플랜 (프리뷰) | 전체 |
| 브라우저 통합 | Claude Code 사용자 | Chrome |
| Dispatch 연동 | Claude Pro, Max | iOS + macOS |

## 무엇이 의미 있는가

Computer Use의 핵심은 **커넥터 없이도 모든 소프트웨어를 자동화할 수 있다**는 것이다.

기존 자동화 도구(RPA, Selenium 등)는 미리 정의된 절차를 따른다. 화면이 조금 바뀌면 스크립트가 깨진다. Claude의 Computer Use는 사람처럼 화면을 이해하고 맥락에 맞게 행동한다. 팝업이 뜨면 처리하고, 레이아웃이 바뀌어도 원하는 버튼을 찾아낸다.

개발자 관점에서 Claude Code의 Auto Mode와 브라우저 통합은 "터미널에서 코드 작성"의 범위를 "터미널에서 전체 개발 워크플로우"로 확장한다. 코드를 짜고, 테스트하고, PR을 올리고, 배포 상태를 확인하는 과정을 이어서 수행할 수 있다.

---

AI가 텍스트를 생성하는 단계에서, 컴퓨터를 직접 조작하는 단계로 넘어가고 있다. 아직 연구 프리뷰지만 방향은 분명하다. AI 에이전트가 우리 대신 화면을 보고, 클릭하고, 작업을 완수하는 시대가 시작됐다.
