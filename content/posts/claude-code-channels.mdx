2026년 3월 20일 Anthropic이 **Claude Code Channels**를 공개했다. Telegram이나 Discord에서 메시지를 보내면 로컬 머신에서 실행 중인 Claude Code 세션이 깨어나 코드를 짜고, 테스트를 돌리고, git 작업을 처리한 뒤 같은 채팅방에 결과를 돌려준다.

## 어떤 문제를 해결하는가

Claude Code는 강력하지만 터미널 앞에 있어야 쓸 수 있었다. 외출 중에 "아, 저 함수 수정해야 하는데"라는 생각이 들어도 노트북이 없으면 방법이 없었다. Channels는 이 제약을 없앤다.

```
외출 중 (Telegram/Discord)
    "로그인 API의 토큰 만료 처리 추가해줘"
        ↓
로컬 Claude Code 세션 (백그라운드 실행 중)
    파일시스템 접근 → 코드 수정 → 테스트 실행 → git commit
        ↓
같은 채팅방
    "완료했습니다. auth/token.ts 수정, 테스트 3개 추가, PR #47 생성했습니다."
```

단순한 원격 채팅이 아니다. Claude Code가 가진 **전체 로컬 환경** — 파일시스템, MCP 도구, git, 터미널 — 을 그대로 활용한다.

## 기술 기반: MCP 채널 프로토콜

Channels는 Anthropic이 2024년에 발표한 **MCP(Model Context Protocol)** 위에 구축됐다. MCP를 "AI와 외부 서비스를 연결하는 범용 포트"로 쓰는 방식이다.

```
Telegram 봇 ──┐
              │ MCP Channel Protocol
Discord 봇  ──┤
              │
    Claude Code 세션 (로컬)
        ├── 파일시스템
        ├── MCP 도구들
        ├── git
        └── 터미널
```

플러그인 아키텍처로 설계되어 있어서 Telegram, Discord 외에 커스텀 채널도 구현할 수 있다. Anthropic이 공개한 채널 레퍼런스를 보면 MCP 채널 프로토콜을 구현한 어떤 서비스도 Claude Code 세션과 연결할 수 있다.

## 사전 요구사항

- **Claude Code v2.1.80 이상**
- **Bun 설치** — MCP 서버가 Bun 위에서 실행됨
- **claude.ai 로그인** — API 키 인증은 지원하지 않음

```bash
# Bun 설치 (없다면)
curl -fsSL https://bun.sh/install | bash

# Claude Code 버전 확인
claude --version

# claude.ai 로그인 상태 확인
claude auth status
```

## Telegram 연동 설정

### 1단계: Telegram 봇 생성

Telegram에서 **BotFather**를 열고 봇을 만든다.

```
BotFather에게:
/newbot

→ 봇 이름 입력 (예: "My Dev Agent")
→ 봇 유저명 입력 (예: "mydevagent_bot", _bot으로 끝나야 함)
→ 토큰 발급: 1234567890:AAHxxxxxx...
```

발급받은 토큰을 복사해 둔다.

### 2단계: 플러그인 설치 및 토큰 설정

Claude Code 안에서 실행한다.

```bash
# 플러그인 설치
/plugin install telegram@claude-plugins-official

# 봇 토큰 설정
/telegram:configure <발급받은-봇-토큰>
```

토큰은 `~/.claude/channels/telegram/.env`에 저장된다.

### 3단계: Channels 모드로 Claude Code 시작

```bash
claude --channels plugin:telegram@claude-plugins-official
```

### 4단계: 봇과 페어링

Telegram에서 방금 만든 봇에게 아무 메시지나 보낸다. 봇이 **페어링 코드**를 응답한다.

```
봇: 페어링 코드: 7X9K2M

Claude Code 터미널에서 이 코드를 입력하면 페어링이 완료됩니다.
```

터미널에 코드를 입력하면 연결 완료. 이후 Telegram에서 보내는 메시지가 모두 Claude Code 세션으로 전달된다.

## Discord 연동 설정

### 1단계: Discord 봇 애플리케이션 생성

[Discord Developer Portal](https://discord.com/developers/applications)에 접속한다.

1. **New Application** → 이름 입력
2. **Bot** 탭 → **Add Bot** → **Reset Token** → 토큰 복사
3. **Privileged Gateway Intents** → **Message Content Intent** 활성화 (필수)

### 2단계: 봇을 서버에 초대

OAuth2 → URL Generator에서:
- Scopes: `bot`
- Bot Permissions: `Send Messages`, `Read Message History`

생성된 URL로 접속해 원하는 서버에 봇 초대.

### 3단계: 플러그인 설치 및 설정

```bash
/plugin install discord@claude-plugins-official
/discord:configure <봇-토큰>
```

### 4단계: Channels 모드로 실행

```bash
claude --channels plugin:discord@claude-plugins-official
```

Discord의 봇이 있는 채널에서 메시지를 보내면 Claude Code가 응답한다.

## 여러 채널 동시 실행

Telegram과 Discord를 동시에 연결할 수 있다.

```bash
claude --channels plugin:telegram@claude-plugins-official,plugin:discord@claude-plugins-official
```

## 로컬 테스트: Fakechat

봇 토큰 없이 채널 기능을 먼저 확인하고 싶다면 공식 지원 데모 채널을 사용한다.

```bash
/plugin install fakechat@claude-plugins-official
claude --channels plugin:fakechat@claude-plugins-official
```

`localhost:8787`에 로컬 채팅 UI가 열린다. Bun과 Claude Code 설정이 올바른지 확인하기 좋은 출발점이다.

## 보안 설계

보안은 두 층으로 구성된다.

**발신자 허용 목록(Allowlist)** — 페어링 시 등록된 사용자 ID만 메시지가 전달된다. 허용 목록에 없는 ID에서 온 메시지는 조용히 무시된다.

```bash
# 추가 사용자 허용 (팀원 등)
/telegram:allow <telegram-user-id>
/discord:allow <discord-user-id>
```

**봇 토큰 보안** — 봇 토큰은 Claude Code 세션의 인증 자격증명이다. SSH 키와 동일하게 취급해야 한다. 유출되면 누군가 당신의 로컬 머신에서 Claude Code를 원격으로 실행할 수 있다.

```bash
# 토큰은 환경변수로도 주입 가능
export TELEGRAM_BOT_TOKEN=...
export DISCORD_BOT_TOKEN=...
claude --channels plugin:telegram@claude-plugins-official
```

## 실전 활용 시나리오

**외출 중 버그 대응**
```
"프로덕션 오류 로그 확인해서 원인 파악하고 hotfix 브랜치에 수정해줘"
→ 로그 분석 → 수정 → PR 생성 → 슬랙 알림 (MCP)
```

**백그라운드 장시간 작업 모니터링**
```
"API 통합 테스트 전체 돌리고 결과 알려줘"
→ 테스트 실행 (수분 소요) → 완료 시 결과 메시지
```

**팀 협업 (Discord 서버 채널)**
```
팀 Discord 채널에서 특정 채널을 "AI 에이전트 채널"로 지정
팀원이 "이슈 #234 구현해줘" → Claude Code가 처리 → 팀 전체에 결과 공유
```

## 커스텀 채널 구현

MCP 채널 프로토콜을 구현하면 Telegram, Discord 외에도 연결할 수 있다. Slack, WhatsApp, 사내 메신저, 웹훅 등 어떤 플랫폼도 이론적으로 가능하다.

```typescript
// 커스텀 채널의 기본 구조 (MCP Channel Protocol)
interface ChannelPlugin {
  name: string
  configure(options: Record<string, string>): Promise<void>
  startListening(onMessage: (msg: ChannelMessage) => void): Promise<void>
  send(message: ChannelMessage): Promise<void>
  stop(): Promise<void>
}
```

공식 Channels 레퍼런스 문서에 전체 프로토콜 스펙이 공개되어 있다.

---

Claude Code Channels는 "AI 코딩 에이전트는 터미널 앞에 있을 때만 쓸 수 있다"는 제약을 없앤다. 로컬 환경의 전체 파워를 유지하면서, 어디서든 메시지 하나로 에이전트를 작동시킬 수 있다. MCP 기반의 플러그인 아키텍처가 확장성을 보장하므로, 앞으로 더 많은 플랫폼 지원과 커뮤니티 플러그인이 나올 것으로 보인다.
