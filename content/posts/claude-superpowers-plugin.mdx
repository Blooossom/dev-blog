Claude Code는 기본 기능만으로도 강력하지만, Superpowers 플러그인을 쓰면 소프트웨어 개발의 전체 사이클을 체계적으로 다룰 수 있게 된다. 단순히 AI에게 코드를 요청하는 것이 아니라, 브레인스토밍부터 설계, 구현, 리뷰, 배포까지 각 단계에 맞는 워크플로우를 제공한다.

## Superpowers란

Superpowers는 Claude Code를 위한 **스킬(Skill) 시스템**이다. 각 스킬은 특정 작업을 어떻게 접근하고 실행해야 하는지 정의한 지침이다. Claude Code가 해당 스킬을 로드하면 그 지침에 따라 체계적으로 작업을 진행한다.

스킬은 단순한 프롬프트가 아니다. 체크리스트, 결정 흐름도, 품질 기준, 실패 처리 방식까지 담고 있다. "좋은 시니어 개발자라면 이 상황에서 어떻게 할까"를 코드화한 것에 가깝다.

## 핵심 스킬들

### 브레인스토밍 (superpowers:brainstorming)

구현을 시작하기 전에 아이디어를 설계로 발전시킨다.

프로세스:
1. 현재 프로젝트 컨텍스트 탐색
2. 한 번에 하나씩 명확화 질문
3. 2~3가지 접근 방식 제안 (장단점 포함)
4. 설계 섹션별 검토 및 승인
5. 설계 문서 저장 (`docs/superpowers/specs/`)
6. 스펙 검토 후 구현 계획 수립으로 전환

핵심 원칙: **구현 코드 작성 전에 반드시 설계를 먼저 승인받는다.** 아무리 간단해 보이는 작업도 이 과정을 건너뛰지 않는다.

### 구현 계획 작성 (superpowers:writing-plans)

승인된 설계를 실행 가능한 태스크 목록으로 변환한다.

계획의 특징:
- 태스크당 2~5분 단위로 잘게 쪼갬
- 정확한 파일 경로, 완성된 코드 포함
- TDD: 테스트 작성 → 실패 확인 → 구현 → 통과 확인 → 커밋
- 계획 문서 저장 (`docs/superpowers/plans/`)

```markdown
### Task 1: 사용자 스키마 정의

**Files:**
- Create: `src/schemas/user.ts`
- Test: `src/schemas/__tests__/user.test.ts`

- [ ] 실패하는 테스트 작성
- [ ] 테스트 실행 확인 (FAIL)
- [ ] 스키마 구현
- [ ] 테스트 통과 확인
- [ ] 커밋
```

### 서브에이전트 기반 개발 (superpowers:subagent-driven-development)

계획의 각 태스크를 독립된 서브에이전트가 실행한다.

워크플로우:
```
태스크 → 구현 에이전트 → 스펙 준수 리뷰 → 코드 품질 리뷰 → 완료
```

왜 서브에이전트인가:
- **컨텍스트 오염 방지**: 각 태스크가 깨끗한 컨텍스트에서 시작
- **집중된 작업**: 에이전트가 해당 태스크에만 집중
- **리뷰 자동화**: 구현 후 스펙 준수 여부, 코드 품질 순서로 두 단계 검토

리뷰는 두 단계다:
1. **스펙 리뷰어**: "요청한 것이 구현됐는가? 불필요한 것이 추가됐는가?"
2. **코드 품질 리뷰어**: "코드가 잘 작성됐는가? 유지보수 가능한가?"

### TDD (superpowers:test-driven-development)

테스트 주도 개발을 체계적으로 따른다.

```
Red (실패하는 테스트) → Green (최소 구현) → Refactor (리팩토링)
```

단순히 "테스트 먼저 써라"가 아니다. 각 단계에서 무엇을 어떻게 해야 하는지, 어떤 테스트가 좋은 테스트인지 구체적인 기준을 제공한다.

### 체계적 디버깅 (superpowers:systematic-debugging)

버그를 만났을 때 추측으로 접근하지 않는다.

```
증상 분류 → 파일 추적 → 가설 수립 → 검증 → 수정
```

같은 실패를 반복하지 않도록 디버깅 과정에서 발견한 원인을 기록한다.

### 개발 브랜치 마무리 (superpowers:finishing-a-development-branch)

구현 완료 후 통합 방법을 결정한다.

옵션 제시:
- 현재 브랜치에서 직접 merge
- PR 생성
- 임시 코드 정리

각 옵션의 적합한 상황을 안내한다.

## 스킬 활성화 방법

```bash
# Claude Code에서 Skill 도구로 호출
/skill superpowers:brainstorming
/skill superpowers:writing-plans
/skill superpowers:test-driven-development
```

또는 자연어로 요청해도 Claude Code가 관련 스킬을 자동으로 판단해서 사용한다.

## 실제 프로젝트에서의 흐름

이 블로그 자체가 Superpowers 워크플로우로 만들어졌다.

1. **브레인스토밍**: 어떤 기술 스택, 어떤 디자인 시스템, 어떤 기능을 만들지 대화로 구체화
2. **설계 문서**: 결정된 사항을 `docs/superpowers/specs/` 에 저장
3. **구현 계획**: 16개 태스크로 분해, 각 태스크에 정확한 파일 경로와 코드 포함
4. **서브에이전트 실행**: 각 태스크를 독립 에이전트가 TDD로 구현
5. **리뷰 루프**: 스펙 준수 → 코드 품질, 문제 발견 시 수정 후 재검토

각 단계에서 발견된 문제들 — Subscribe 버튼 누락, border 규칙 위반, 이중 파일 읽기 — 이 리뷰 단계에서 잡혔다. 스킬이 없었다면 임의로 코드를 작성하다가 나중에 발견했을 문제들이다.

## Superpowers의 핵심 철학

**YAGNI** — 지금 필요한 것만 만든다. 미래를 위한 추상화나 확장성은 요구되기 전까지 만들지 않는다.

**TDD** — 테스트가 먼저다. 구현보다 테스트가 설계를 이끈다.

**빈번한 커밋** — 작은 단위로 자주 커밋한다. 각 태스크 완료마다 커밋.

**단일 진실 공급원** — 스펙 문서, 구현 계획, 실제 코드가 일치해야 한다.

---

Superpowers는 "AI에게 코드 짜달라"는 접근에서 "AI와 함께 소프트웨어를 개발한다"로 패러다임을 바꾼다. AI가 구현을 맡고, 사람이 설계와 판단을 맡는 협업 구조다. 그리고 이 협업이 제대로 작동하려면 단계별로 명확한 프로세스가 필요하다. Superpowers 스킬이 그 프로세스를 제공한다.
