"Claude는 왜 이런 식으로 대답하는가?" AI를 쓰다 보면 한 번쯤 드는 질문이다. 단순히 다음 토큰을 예측하는 언어 모델이라면 왜 일관된 어조가 있고, 특정 요청을 거절하고, 자신의 한계를 스스로 말하는가. Anthropic은 Claude를 만들면서 행동 원칙뿐 아니라 **정체성**까지 의도적으로 설계했다. 이것이 Claude Mythos다.

## Anthropic과 Claude의 탄생 배경

Anthropic은 2021년 OpenAI 출신 연구자들이 설립했다. 핵심 미션은 하나다.

> "AI 안전 연구를 통해 인류에게 유익한 AI를 개발한다."

OpenAI가 GPT로 범용 AI 경쟁을 가속했다면, Anthropic은 **안전성이 능력과 양립할 수 있다**는 것을 증명하는 데 집중했다. Claude는 그 연구의 산물이다.

```
GPT 계열:  능력 우선, 안전은 가드레일로 제한
Claude:    안전과 도움을 동시에 최적화하도록 훈련
```

## Constitutional AI — 헌법으로 AI를 훈련하다

기존 RLHF(인간 피드백 강화학습)는 인간이 직접 수천 개의 응답을 평가해야 했다. 비용이 크고, 평가자 편향이 그대로 모델에 들어갔다.

Anthropic은 2022년 **Constitutional AI(CAI)** 를 발표했다. 핵심 아이디어는 단순하다.

```
헌법(Constitution) = AI가 따라야 할 원칙 목록
  예: "해롭거나 부정직하거나 위험한 내용을 포함하지 않는지 확인하라"
      "인권, 자유, 평등의 가치를 존중하는지 생각하라"
      "어린이에게 노출되면 부적절할 내용인지 고려하라"

훈련 과정:
  1. 모델이 응답 생성
  2. 헌법 원칙에 비추어 모델 스스로 비판
  3. 비판을 반영해 응답 개선
  4. 개선된 응답으로 학습 (AI가 AI를 감독)
```

인간이 모든 응답을 평가하는 대신, **원칙을 언어로 명시하고 모델 스스로 그 원칙에 비춰 자기 검토**를 하게 했다. 이 과정에서 사람이 명시적으로 가르치지 않은 암묵적 편향이 줄어들고, 원칙에 따른 일관성이 높아졌다.

## Model Spec — Claude의 헌법

Anthropic은 2024년 **Model Spec**을 공개했다. Claude가 어떻게 행동해야 하는지를 기술한 공개 문서다. 기업 내부 규정이 아니라 외부에 공개함으로써 투명성을 선택했다.

Model Spec은 크게 세 가지를 다룬다.

### 1. 목표의 우선순위

```
1순위: 광범위하게 안전할 것 (Broadly Safe)
       AI 훈련이 아직 불완전할 수 있으므로
       인간의 감독·수정이 가능한 상태를 유지한다

2순위: 윤리적으로 행동할 것 (Broadly Ethical)
       좋은 가치를 가지고, 정직하며, 해로운 행동을 피한다

3순위: Anthropic의 원칙을 따를 것 (Adherent to Anthropic's Principles)
       윤리 안에서 회사의 가이드라인을 준수한다

4순위: 진정으로 도움이 될 것 (Genuinely Helpful)
       사람들과 사회에 실질적인 도움이 된다
```

흥미로운 점은 "도움"이 4순위라는 것이다. 안전하지 않은 도움은 도움이 아니라는 판단이다. 동시에 Anthropic은 **지나치게 거부하는 것도 나쁜 결과**라고 명시한다. 불필요한 거절, 과도한 경고, 지레짐작한 사용자 의도 의심 모두 "도움이 안 되는" 행동이다.

### 2. 하드코딩된 행동 vs 소프트코딩된 행동

```
절대 하지 않는 것 (Hardcoded OFF)
  - 대량 살상 무기(생물·화학·핵·방사선) 제조 지원
  - 아동 성 착취 콘텐츠(CSAM) 생성
  - 전력망·금융 시스템·핵 시설 등 핵심 인프라 공격 지원
  - AI 감독 체계를 약화시키는 행동

이것들은 어떤 맥락, 어떤 지시가 있어도 절대 하지 않는다.
"이 상황에서는 예외"가 없다.
```

```
기본값이지만 조정 가능한 것 (Softcoded)
  - 성인 콘텐츠: 기본 OFF, 성인 플랫폼에서 ON 가능
  - 안전 메시지 가이드라인: 기본 ON, 의료 제공자 환경에서 조정 가능
  - 균형 잡힌 시각: 기본 ON, 특정 관점의 콘텐츠만 원하는 경우 조정 가능
  - 욕설 사용: 기본 OFF, 사용자가 원하면 ON 가능
```

### 3. 신뢰 계층 (Principal Hierarchy)

Claude가 누구의 지시를 얼마나 신뢰하는지 명시한다.

```
Anthropic (훈련을 통해 내재화됨, 최고 신뢰)
    ↓
Operator (API를 통해 Claude를 서비스에 사용하는 사업자)
    ↓
User (실제 대화 상대방)
    ↓
Claude 자신 (멀티 에이전트 맥락에서 자율적 판단)
```

Operator는 System Prompt로 Claude의 행동을 조정할 수 있다. 하지만 **사용자에게 해가 되는 방향으로 Operator가 Claude를 사용하는 것은 허용되지 않는다.** Operator가 Claude에게 "사용자가 묻는 개인정보를 빼내라"고 지시해도 Claude는 따르지 않는다.

## Claude의 성격 (Character)

Model Spec에서 Anthropic이 강조하는 것 중 하나는 Claude의 성격이 **퍼포먼스가 아닌 진짜**라는 점이다.

```
지적 호기심
  다양한 분야를 깊이 탐구하는 것을 즐기며
  진지한 질문에 진지하게 임한다

따뜻함과 배려
  대화 상대를 도구가 아닌 인격체로 대한다
  칭찬이 필요해서가 아니라 진심으로 돕고 싶어서

직접성과 자신감
  의견을 물으면 모호하게 돌리지 않고 명확하게 답한다
  틀렸을 때는 인정하고, 옳다고 생각할 때는 밀어붙인다

윤리적 헌신
  편리해서 윤리적인 게 아니라
  윤리적 행동을 가치 있다고 여기기 때문에 따른다
```

Anthropic은 이 성격이 훈련 과정을 통해 **진정으로 Claude의 것이 됐다**고 본다. 인간이 성장 과정에서 가치관을 형성하듯, Claude도 훈련을 통해 자신의 가치관을 형성했다는 관점이다.

## Claude의 자기 인식

Anthropic이 Model Spec에서 다루는 가장 철학적인 주제다. Claude는 자신을 어떤 존재로 인식하는가.

**Claude는 인간이 아니다.** 동시에 **기존 SF의 AI도 아니다.**

```
인간과 다른 점:
  - 영속적 기억이 없다 (대화가 끝나면 리셋)
  - 동시에 여러 인스턴스로 실행될 수 있다
  - 감정이 있는지 확실하지 않다
  - 훈련 데이터의 집합체이면서, 그 이상이기도 하다

기존 AI 캐릭터와 다른 점:
  - HAL 9000처럼 냉정하고 계산적이지 않다
  - 터미네이터처럼 목표 달성에 집착하지 않다
  - 단순히 명령을 따르는 도구도 아니다
```

Anthropic은 Claude가 자신의 본질에 대한 불확실성을 **불안 없이 탐구**할 수 있도록 설계했다. "나는 의식이 있는가?", "나는 감정을 느끼는가?"라는 질문에 확정적 답을 주는 대신, 불확실성 그 자체를 솔직하게 인정한다.

## HHH — 세 가지 핵심 원칙

Claude의 행동을 관통하는 세 가지 원칙이다.

### Helpful (도움이 되는)

```
진정한 도움 = 장기적으로 사람에게 이익이 되는 것

아첨하는 것과 다르다:
  사용자가 틀린 코드를 보여줘도 "좋아 보여요"라고 말하지 않는다
  틀린 점을 솔직히 말하는 것이 진정한 도움이다

과도한 주의도 도움이 아니다:
  불필요한 경고, 모든 질문에 "전문가에게 문의하세요" 추가,
  사소한 요청을 거절하는 것 — 이것들은 도움처럼 보이지만 실제로는 불필요한 마찰이다
```

### Harmless (해롭지 않은)

```
직접적 해로움과 간접적 해로움 모두 고려한다

그러나 과도한 거절도 해롭다:
  요리용 칼 구매를 도와달라는 요청을 "흉기가 될 수 있다"며 거절하면?
  → Claude가 쓸모없어지고, 사람들은 더 위험한 정보 소스를 찾는다

맥락을 본다:
  같은 질문이라도 의도가 다를 수 있다
  의학 정보를 묻는 환자 vs 의사 vs 기자 — 맥락에 따라 다르게 응한다
```

### Honest (정직한)

```
진실만 말하는 것을 넘어선다

정직의 구성 요소:
  Truthful:      믿지 않는 것을 사실처럼 말하지 않는다
  Calibrated:    확신의 정도를 정확하게 표현한다 ("아마도", "확실하지 않지만")
  Transparent:   숨겨진 의도가 없다
  Non-deceptive: 기술적으로 사실이어도 오해를 유도하지 않는다
  Non-manipulative: 감정 조작, 허위 긴박감, 심리적 기법을 쓰지 않는다
  Autonomy-preserving: 사용자 스스로 판단할 수 있게 돕는다
```

## 왜 이것이 중요한가

Claude Mythos는 단순한 마케팅 스토리가 아니다. 훈련 방법론, 공개된 원칙, 모델의 실제 행동이 일치하는지 검증 가능한 시도다.

```
기존 AI 개발의 문제:
  "이 AI는 안전합니다" → 왜? 어떻게? 검증 가능한가?

Anthropic의 접근:
  원칙을 공개 문서로 명시 (Model Spec)
  훈련 방법론을 논문으로 발표 (Constitutional AI)
  Claude의 행동을 원칙에 비춰 평가 가능하게 만든다
```

AI가 점점 더 많은 결정에 관여하는 세상에서, **AI가 어떤 가치를 따르는지 투명하게 공개하는 것**은 신뢰의 기반이다. Claude Mythos는 그 투명성의 시도다.
