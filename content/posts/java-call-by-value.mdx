자바 면접에서 단골로 등장하는 질문이 있다. "자바는 Call by Value인가, Call by Reference인가?" 정답은 **항상 Call by Value**다. 하지만 객체를 메서드에 넘기면 내부 상태가 바뀌는 경험을 해봤기 때문에 헷갈린다. 이 글에서 그 오해를 정확히 풀어낸다.

## Call by Value vs Call by Reference

### Call by Value

메서드를 호출할 때 **값을 복사해서 전달**한다. 메서드 내부에서 매개변수를 아무리 바꿔도 원본 변수에는 영향이 없다.

### Call by Reference

값이 아닌 **변수의 메모리 주소 자체를 전달**한다. 메서드 내부에서 매개변수를 바꾸면 원본 변수도 함께 바뀐다.

| 구분 | 전달하는 것 | 원본 변경 가능 여부 |
|------|-----------|-------------------|
| Call by Value | 값의 복사본 | 불가 |
| Call by Reference | 변수의 주소 자체 | 가능 |

---

## 자바의 기본형 — 명확한 Call by Value

기본형(primitive type)은 Call by Value가 명백하게 드러난다.

```java
public class Main {
    public static void main(String[] args) {
        int x = 10;
        increment(x);
        System.out.println(x); // 10 — 변하지 않음
    }

    static void increment(int n) {
        n = n + 1; // 복사본을 변경
        System.out.println(n); // 11
    }
}
```

```
main 스택 프레임       increment 스택 프레임
┌───────────┐         ┌───────────┐
│  x = 10   │  복사   │  n = 10   │ → n = 11
└───────────┘  ──────→ └───────────┘
     ↑
  원본 불변
```

`increment` 안에서 `n`을 바꿔도 `main`의 `x`는 전혀 영향받지 않는다. 두 변수는 완전히 독립된 스택 공간에 있다.

## 자바의 참조형 — 오해가 시작되는 지점

객체를 넘기면 상황이 다르게 보인다.

```java
class User {
    String name;
    User(String name) { this.name = name; }
}

public class Main {
    public static void main(String[] args) {
        User user = new User("Alice");
        changeName(user);
        System.out.println(user.name); // "Bob" — 바뀌었다!
    }

    static void changeName(User u) {
        u.name = "Bob"; // 객체 내부 필드를 변경
    }
}
```

객체의 필드가 바뀌었다. 이것만 보면 "자바는 Call by Reference 아닌가?"라고 생각하기 쉽다. **하지만 그렇지 않다.**

### 실제로 무슨 일이 일어났는가

```
main 스택 프레임                힙(Heap)
┌──────────────────┐          ┌──────────────┐
│ user = 0xABCD    │──────→  │ User 객체    │
└──────────────────┘          │ name="Alice" │
                               └──────────────┘
          ↓ 복사 (Call by Value)
changeName 스택 프레임
┌──────────────────┐
│ u    = 0xABCD    │──────→  (같은 힙 객체)
└──────────────────┘
```

`user` 변수가 담고 있는 것은 **객체 자체가 아니라 객체의 힙 주소(참조값)**다. 메서드를 호출할 때 이 **주소값이 복사**되어 전달된다. `u`와 `user`는 서로 다른 변수지만, **같은 힙 주소를 가리키고 있다.** 그래서 `u.name = "Bob"`은 같은 객체의 필드를 수정하게 되고, `user.name`을 읽으면 "Bob"이 나온다.

> **정리:** 전달된 것은 주소값의 복사본이다. 주소 자체가 넘어간 것이 아니다.

## Call by Reference와 결정적으로 다른 증거

진짜 Call by Reference라면 메서드 안에서 매개변수 자체를 다른 객체로 바꿨을 때 원본도 바뀌어야 한다. 자바에서 실험해보자.

```java
public class Main {
    public static void main(String[] args) {
        User user = new User("Alice");
        replace(user);
        System.out.println(user.name); // "Alice" — 바뀌지 않음!
    }

    static void replace(User u) {
        u = new User("Charlie"); // u가 새 객체를 가리키게 함
        System.out.println(u.name); // "Charlie"
    }
}
```

```
main 스택 프레임                힙
┌──────────────────┐          ┌──────────────┐
│ user = 0xABCD    │──────→  │ User("Alice")│  ← 원본 불변
└──────────────────┘          └──────────────┘

replace 호출 전:
┌──────────────────┐
│ u    = 0xABCD    │──────→  (같은 객체)
└──────────────────┘

u = new User("Charlie") 실행 후:
┌──────────────────┐          ┌────────────────┐
│ u    = 0xEF01    │──────→  │ User("Charlie")│ (새 객체)
└──────────────────┘          └────────────────┘
                               main의 user는 여전히 0xABCD
```

`replace` 안에서 `u`를 새 객체로 교체해도 `main`의 `user`는 여전히 `"Alice"`다. 복사된 주소값만 바뀌었을 뿐, 원본 변수(`user`)가 가리키는 곳은 그대로다.

**이것이 자바가 Call by Reference가 아닌 결정적 증거다.**

## String이 불변인 이유와의 연결

```java
public class Main {
    public static void main(String[] args) {
        String s = "hello";
        modify(s);
        System.out.println(s); // "hello" — 변하지 않음
    }

    static void modify(String str) {
        str = str + " world"; // 새 String 객체 생성
    }
}
```

`String`은 불변(immutable) 객체다. `str + " world"`는 기존 문자열을 바꾸는 것이 아니라 새 `String` 객체를 만들어 `str`에 할당한다. 원본 `s`는 여전히 `"hello"`를 가리킨다. 이 역시 Call by Value 동작과 일치한다.

## 배열도 마찬가지다

배열도 참조형이므로 동일한 규칙이 적용된다.

```java
public class Main {
    public static void main(String[] args) {
        int[] arr = {1, 2, 3};
        modifyElement(arr);
        System.out.println(arr[0]); // 100 — 내부 요소는 변경됨

        reassign(arr);
        System.out.println(arr[0]); // 100 — 배열 자체 교체는 반영 안 됨
    }

    static void modifyElement(int[] a) {
        a[0] = 100; // 같은 배열 객체의 요소를 수정
    }

    static void reassign(int[] a) {
        a = new int[]{9, 9, 9}; // a가 새 배열을 가리키게 함 (원본 무관)
    }
}
```

## 핵심 정리

| 상황 | 동작 | 원본 변경 |
|------|------|---------|
| 기본형 전달 | 값 복사 | ❌ |
| 객체 전달 후 필드 수정 | 주소값 복사 → 같은 객체 접근 | ✅ (객체 내부) |
| 객체 전달 후 재할당 | 복사된 주소값만 바뀜 | ❌ |

자바에서 메서드에 전달되는 것은 언제나 **값의 복사본**이다. 기본형이면 값 자체가, 참조형이면 힙 주소값이 복사된다. 참조형을 넘길 때 객체 내부를 바꿀 수 있는 것은 두 변수가 같은 힙 객체를 가리키기 때문이지, 주소 자체가 공유되기 때문이 아니다.
