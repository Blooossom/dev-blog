TypeScript 유틸리티 타입은 기존 타입을 변환하는 도구다. 잘 쓰면 타입 중복을 없애고 코드의 의도를 명확히 표현할 수 있다. 자주 쓰는 것부터 원리까지 정리해봤다.

## 기본 유틸리티 타입

### Partial\<T\>

모든 프로퍼티를 선택적으로 만든다.

```typescript
interface User {
  id: number
  name: string
  email: string
  role: 'admin' | 'user'
}

// 업데이트 API — 일부 필드만 바꿀 수 있다
function updateUser(id: number, updates: Partial<User>) {
  // updates.name, updates.email 등이 모두 optional
}

updateUser(1, { name: '홍길동' })         // ✅
updateUser(1, { name: '홍길동', role: 'admin' }) // ✅
```

### Required\<T\>

모든 프로퍼티를 필수로 만든다. `Partial`의 반대.

```typescript
interface Config {
  apiUrl?: string
  timeout?: number
  retries?: number
}

// 기본값이 채워진 후에는 모든 필드가 존재함을 보장
function applyDefaults(config: Config): Required<Config> {
  return {
    apiUrl: config.apiUrl ?? 'https://api.example.com',
    timeout: config.timeout ?? 5000,
    retries: config.retries ?? 3,
  }
}
```

### Readonly\<T\>

모든 프로퍼티를 읽기 전용으로 만든다.

```typescript
interface Point {
  x: number
  y: number
}

const origin: Readonly<Point> = { x: 0, y: 0 }
origin.x = 1 // ❌ 에러: Cannot assign to 'x' because it is a read-only property
```

### Pick\<T, K\>

타입에서 원하는 프로퍼티만 선택한다.

```typescript
interface User {
  id: number
  name: string
  email: string
  password: string
  role: 'admin' | 'user'
}

// API 응답에서 민감 정보 제외
type PublicUser = Pick<User, 'id' | 'name' | 'email'>
// → { id: number; name: string; email: string }

// 카드 컴포넌트에 필요한 것만
type UserCardProps = Pick<User, 'id' | 'name'>
```

### Omit\<T, K\>

타입에서 특정 프로퍼티를 제거한다. `Pick`의 반대.

```typescript
// 새 유저 생성 시 id는 서버가 생성 — id 제외
type CreateUserDto = Omit<User, 'id'>

// 비밀번호 제외한 안전한 타입
type SafeUser = Omit<User, 'password'>
```

### Record\<K, V\>

키-값 맵 타입을 만든다.

```typescript
type Role = 'admin' | 'editor' | 'viewer'

// 각 역할의 권한 목록
const permissions: Record<Role, string[]> = {
  admin: ['read', 'write', 'delete'],
  editor: ['read', 'write'],
  viewer: ['read'],
}

// 동적 키 맵
type CountryCode = string
const countryNames: Record<CountryCode, string> = {
  KR: '대한민국',
  US: '미국',
  JP: '일본',
}
```

### Exclude\<T, U\>와 Extract\<T, U\>

유니온 타입에서 제거하거나 추출한다.

```typescript
type Status = 'pending' | 'active' | 'inactive' | 'deleted'

// 삭제된 상태 제외
type ActiveStatus = Exclude<Status, 'deleted'>
// → 'pending' | 'active' | 'inactive'

// 활성 상태만
type OnlyActive = Extract<Status, 'active' | 'inactive'>
// → 'active' | 'inactive'
```

### NonNullable\<T\>

`null`과 `undefined`를 제거한다.

```typescript
type MaybeString = string | null | undefined

type DefinitelyString = NonNullable<MaybeString>
// → string

// 실용적인 사용
function processValue(value: string | null | undefined): string {
  const safe: NonNullable<typeof value> = value ?? ''
  return safe.toUpperCase()
}
```

## 함수 관련 유틸리티 타입

### ReturnType\<T\>

함수의 반환 타입을 추출한다.

```typescript
function getUser() {
  return { id: 1, name: '홍길동', email: 'hong@example.com' }
}

type User = ReturnType<typeof getUser>
// → { id: number; name: string; email: string }

// 비동기 함수
async function fetchUser() {
  const response = await fetch('/api/user')
  return response.json() as Promise<User>
}

type FetchUserReturn = Awaited<ReturnType<typeof fetchUser>>
// → User
```

### Parameters\<T\>

함수의 파라미터 타입을 튜플로 추출한다.

```typescript
function createOrder(userId: number, items: string[], coupon?: string) {
  // ...
}

type CreateOrderParams = Parameters<typeof createOrder>
// → [userId: number, items: string[], coupon?: string]

// 특정 파라미터만
type FirstParam = Parameters<typeof createOrder>[0] // number
```

### Awaited\<T\>

`Promise`를 언래핑한다.

```typescript
type WrappedString = Promise<Promise<string>>

type Unwrapped = Awaited<WrappedString>
// → string (중첩 Promise도 재귀적으로 언래핑)
```

## 조건부 타입과 infer

고급 유틸리티 타입의 원리를 이해하려면 조건부 타입과 `infer`를 알아야 한다.

### 조건부 타입

```typescript
type IsString<T> = T extends string ? 'yes' : 'no'

type A = IsString<string> // 'yes'
type B = IsString<number> // 'no'
```

유니온 타입에 적용하면 분배(distributive)된다.

```typescript
type ToArray<T> = T extends any ? T[] : never

type Result = ToArray<string | number>
// → string[] | number[]
// (string | number)[]이 아님에 주의
```

### infer

조건부 타입 안에서 타입을 추론해 변수처럼 사용한다.

```typescript
// 배열 요소 타입 추출
type ElementType<T> = T extends (infer E)[] ? E : never

type Str = ElementType<string[]>   // string
type Num = ElementType<number[]>   // number
type Never = ElementType<string>   // never (배열이 아니므로)

// Promise 언래핑 (Awaited의 원리)
type Unwrap<T> = T extends Promise<infer U> ? Unwrap<U> : T

// 함수 반환 타입 (ReturnType의 원리)
type MyReturnType<T> = T extends (...args: any[]) => infer R ? R : never
```

## 실전 패턴

### API 응답 타입에서 도메인 타입 분리

```typescript
// API 응답 스키마 (snake_case)
interface ApiUserResponse {
  user_id: number
  full_name: string
  email_address: string
  created_at: string
}

// 도메인 타입 (camelCase)
interface User {
  id: number
  name: string
  email: string
  createdAt: Date
}

// 변환 함수
function mapUser(api: ApiUserResponse): User {
  return {
    id: api.user_id,
    name: api.full_name,
    email: api.email_address,
    createdAt: new Date(api.created_at),
  }
}
```

### 폼 상태 타입

```typescript
interface ProductForm {
  name: string
  price: number
  category: string
  description: string
}

// 폼 필드별 에러 메시지
type FormErrors<T> = Partial<Record<keyof T, string>>

const errors: FormErrors<ProductForm> = {
  name: '이름을 입력해주세요',
  price: '0보다 큰 값을 입력해주세요',
}

// 특정 필드만 수정 가능한 타입
type EditableFields = Pick<ProductForm, 'name' | 'description'>
```

### 이벤트 핸들러 타입

```typescript
type EventMap = {
  click: MouseEvent
  keydown: KeyboardEvent
  submit: SubmitEvent
}

type EventHandler<K extends keyof EventMap> = (event: EventMap[K]) => void

function addHandler<K extends keyof EventMap>(
  element: HTMLElement,
  event: K,
  handler: EventHandler<K>
) {
  element.addEventListener(event, handler as EventListener)
}

// 타입 안전하게 이벤트 핸들러 추가
addHandler(button, 'click', (e) => {
  // e는 MouseEvent로 추론됨
  console.log(e.clientX, e.clientY)
})
```

### Deep Readonly

내장 `Readonly`는 얕은(shallow) 불변이다. 중첩 객체까지 불변으로 만들려면 직접 구현해야 한다.

```typescript
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object ? DeepReadonly<T[K]> : T[K]
}

interface Config {
  database: {
    host: string
    port: number
  }
  cache: {
    ttl: number
  }
}

const config: DeepReadonly<Config> = {
  database: { host: 'localhost', port: 5432 },
  cache: { ttl: 3600 },
}

config.database.host = 'prod.example.com' // ❌ 에러
```

---

유틸리티 타입을 잘 활용하면 타입 정의의 중복이 줄어들고 변경에 강해진다. `User` 타입이 바뀌면 `Partial<User>`, `Pick<User, ...>`, `Omit<User, ...>`는 자동으로 따라 바뀐다. 처음에는 외워서 쓰다 보면 자연스럽게 조건부 타입과 `infer`까지 이어진다.
