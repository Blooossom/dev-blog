PostgreSQL에 접속할 때 "password authentication failed"나 "no pg_hba.conf entry" 오류를 만난 경험이 있을 것이다. 이 오류의 원인은 대부분 `pg_hba.conf` 파일 설정에 있다. 이 파일은 **누가, 어디서, 어떤 방식으로** PostgreSQL에 접속할 수 있는지를 결정한다.

## pg_hba.conf 파일 구조

HBA는 Host-Based Authentication의 약자다. 파일 경로는 PostgreSQL 데이터 디렉터리 안에 있다.

```bash
# 위치 확인
psql -U postgres -c "SHOW hba_file;"
# /etc/postgresql/15/main/pg_hba.conf (Ubuntu)
# /var/lib/pgsql/data/pg_hba.conf (RHEL)
```

각 행은 하나의 규칙을 나타낸다.

```
# TYPE  DATABASE  USER     ADDRESS         METHOD
local   all       all                      peer
host    all       all      127.0.0.1/32    scram-sha-256
host    all       all      ::1/128         scram-sha-256
host    mydb      appuser  10.0.0.0/8      scram-sha-256
```

| 필드 | 설명 |
|------|------|
| TYPE | 연결 유형 (local, host, hostssl, hostnossl) |
| DATABASE | 대상 데이터베이스 (all, 특정 DB 이름) |
| USER | 대상 사용자 (all, 특정 사용자) |
| ADDRESS | 클라이언트 IP/CIDR (local 타입은 생략) |
| METHOD | 인증 방식 |

**규칙은 위에서 아래로 순서대로 검사**하며 첫 번째로 매칭되는 규칙이 적용된다.

---

## 연결 타입 (TYPE)

| 타입 | 설명 |
|------|------|
| `local` | Unix 도메인 소켓 연결 (같은 서버 내) |
| `host` | TCP/IP 연결 (SSL 여부 무관) |
| `hostssl` | SSL을 사용하는 TCP/IP 연결만 허용 |
| `hostnossl` | SSL을 사용하지 않는 TCP/IP 연결만 허용 |

---

## 인증 방식 (METHOD)

### trust

**아무런 인증 없이 접속을 허용**한다. 비밀번호를 묻지 않는다.

```
local   all   postgres   trust
```

로컬 개발 환경에서 편의를 위해 사용하기도 하지만, 운영 환경에서는 절대 사용해서는 안 된다. 해당 OS 계정이 있는 누구든 DB에 접속할 수 있다.

### peer

**OS 사용자 이름과 PostgreSQL 사용자 이름이 일치**하는 경우에만 허용한다. `local` 타입에서만 사용 가능하다.

```
local   all   all   peer
```

```bash
# OS 사용자 postgres로 실행 중이라면
sudo -u postgres psql   # 비밀번호 없이 접속 가능
```

PostgreSQL을 처음 설치하면 `postgres` OS 계정이 생성되고, `local` 연결에 `peer` 인증이 기본 설정된다.

### ident

네트워크 연결에서 `ident` 서버를 통해 OS 사용자를 확인한다. 현재는 거의 사용되지 않는다.

### md5

비밀번호를 **MD5 해시**로 전송한다.

```
host   all   all   0.0.0.0/0   md5
```

MD5는 암호학적으로 취약한 알고리즘이다. 레인보우 테이블 공격에 노출될 수 있고, PostgreSQL 16부터는 공식적으로 사용 중단(deprecated)이 권고된다.

### scram-sha-256

현재 **가장 권장되는 인증 방식**이다. PostgreSQL 10에서 도입되었으며, SCRAM(Salted Challenge Response Authentication Mechanism) 프로토콜을 사용한다.

```
host   all   all   0.0.0.0/0   scram-sha-256
```

md5와 달리:
- 비밀번호 자체가 아닌 해시값만 DB에 저장됨
- 솔트(salt)가 포함되어 레인보우 테이블 공격 방어
- 인증 과정에서 비밀번호가 네트워크에 전송되지 않음 (챌린지-응답 방식)
- SSL 없이도 비밀번호 노출 위험이 낮음

### password

비밀번호를 **평문으로** 전송한다. SSL과 함께 사용하지 않으면 네트워크 도청에 취약하다.

```
# SSL 없이 사용 금지
hostssl   all   all   0.0.0.0/0   password   # SSL 강제 시에만 허용
```

### cert

**클라이언트 SSL 인증서**로 인증한다. 가장 강력하지만 인증서 관리 부담이 있다. `hostssl` 타입에서만 사용 가능하다.

### reject

명시적으로 **접속을 거부**한다. 특정 IP나 사용자를 차단할 때 사용한다.

```
# 특정 IP 차단
host   all   all   192.168.1.50/32   reject

# reject 후 나머지는 허용
host   all   all   0.0.0.0/0         scram-sha-256
```

## 인증 방식 비교

| 방식 | 보안 수준 | 사용 권장 | 특징 |
|------|---------|---------|------|
| `trust` | ❌ 없음 | 로컬 개발만 | 인증 생략 |
| `peer` | ✅ | 로컬 Unix 소켓 | OS 사용자와 매핑 |
| `md5` | ⚠️ 취약 | 사용 지양 | 레거시 |
| `scram-sha-256` | ✅✅ | **강력 권장** | 현재 표준 |
| `password` | ❌ 평문 | SSL 없으면 금지 | 도청 위험 |
| `cert` | ✅✅✅ | 고보안 환경 | 인증서 필요 |
| `reject` | — | 차단 규칙에 활용 | 명시적 거부 |

## 실전 설정 예시

### 로컬 개발 환경

```
# TYPE   DATABASE   USER      ADDRESS          METHOD
local    all        postgres  peer             # 관리자: OS 계정 기반
local    all        all       scram-sha-256    # 일반 사용자: 비밀번호
host     all        all       127.0.0.1/32     scram-sha-256
host     all        all       ::1/128          scram-sha-256
```

### 운영 환경 (앱 서버 + 관리 접근 분리)

```
# TYPE   DATABASE   USER       ADDRESS           METHOD

# 관리자: 로컬 서버에서만 peer 인증
local    all        postgres   peer

# 애플리케이션 계정: 앱 서버 IP 대역에서만 비밀번호 인증
host     mydb       appuser    10.10.0.0/16      scram-sha-256

# DBA 접근: 특정 IP에서만 허용
host     all        dba        10.20.1.100/32    scram-sha-256

# 나머지 모든 접근 차단
host     all        all        0.0.0.0/0         reject
```

### Docker/Kubernetes 환경

컨테이너에서는 `trust`를 내부 네트워크 한정으로 사용하기도 한다.

```
# Docker 내부 네트워크 (172.17.0.0/16)
host   all   all   172.17.0.0/16   scram-sha-256

# 외부 접근은 모두 차단
host   all   all   0.0.0.0/0       reject
```

## 설정 변경 후 적용

```bash
# pg_hba.conf를 수정한 후 PostgreSQL을 재시작하지 않고 적용
psql -U postgres -c "SELECT pg_reload_conf();"

# 또는
sudo systemctl reload postgresql

# 현재 hba 설정 확인 (PostgreSQL 10+)
psql -U postgres -c "TABLE pg_hba_file_rules;"
```

## 기존 사용자 인증 방식 변경

md5에서 scram-sha-256으로 전환 시 비밀번호를 재설정해야 한다. 기존 md5 해시와 scram-sha-256 해시는 호환되지 않는다.

```sql
-- postgresql.conf에서 기본 암호화 방식 변경
-- password_encryption = scram-sha-256

-- 사용자 비밀번호 재설정
ALTER USER appuser WITH PASSWORD 'new-password';
-- 이 시점에 password_encryption 설정에 따라 scram-sha-256으로 저장됨
```

```bash
# 현재 사용자의 인증 방식 확인
psql -U postgres -c "SELECT usename, passwd FROM pg_shadow;"
# passwd가 'scram-sha-256$...'로 시작하면 scram, 'md5...'면 md5
```

---

`pg_hba.conf`의 핵심 원칙은 **최소 권한**이다. 꼭 필요한 사용자만, 꼭 필요한 DB에만, 꼭 필요한 IP에서만 접근을 허용하고, 인증 방식은 `scram-sha-256`을 기본으로 사용해야 한다.
