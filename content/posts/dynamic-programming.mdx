다이나믹 프로그래밍(DP)은 단순히 점화식을 세우는 문제가 아니라, 상태 정의와 중복 구조를 찾는 문제라고 볼 수 있다. 이 차이를 이해하지 못하면 DP 문제 앞에서 항상 막막함을 느끼게 된다.

## DP의 두 가지 조건

DP가 적용 가능한 문제는 반드시 두 가지 특성을 가진다.

| 조건 | 설명 |
|------|------|
| **최적 부분 구조** | 전체 최적해가 부분 최적해로 구성됨 |
| **중복 부분 문제** | 동일한 하위 문제가 여러 번 반복 계산됨 |

이 두 조건이 없으면 DP를 적용해도 효과가 없다. 단순 분할정복과 DP의 차이는 바로 중복 부분 문제의 존재 여부다.

## 상태 정의가 핵심이다

DP 풀이의 첫 단계는 `dp[i]`가 **무엇을 의미하는가**를 정의하는 것이다. 같은 문제도 상태 정의에 따라 점화식이 완전히 달라진다.

### 예시: 최장 증가 부분 수열(LIS)

**상태 정의 A**: `dp[i]` = i번째 원소로 끝나는 LIS의 길이

```java
// O(n^2) 풀이
int[] dp = new int[n];
Arrays.fill(dp, 1);

for (int i = 1; i < n; i++) {
    for (int j = 0; j < i; j++) {
        if (arr[j] < arr[i]) {
            dp[i] = Math.max(dp[i], dp[j] + 1);
        }
    }
}
```

이 정의에서 점화식은 자연스럽게 도출된다. `j`가 `i`보다 앞에 있고 `arr[j]`가 `arr[i]`보다 작으면, `dp[i]`는 `dp[j] + 1`의 최댓값이다.

**상태 정의 B**: `dp[i]` = 길이가 i인 증가 부분 수열의 마지막 원소 최솟값

```java
// O(n log n) 풀이
int[] tails = new int[n];
int size = 0;

for (int num : arr) {
    int pos = Arrays.binarySearch(tails, 0, size, num);
    if (pos < 0) pos = -(pos + 1);
    tails[pos] = num;
    if (pos == size) size++;
}
```

상태 정의 하나가 바뀌었을 뿐인데, 이진 탐색을 활용한 O(n log n) 풀이로 완전히 다른 접근이 가능해진다.

## 중복 구조 발견하기

중복 구조를 찾는 가장 효과적인 방법은 **재귀로 먼저 생각하는 것**이다.

```
f(n) = 문제를 n 크기로 정의
     = 더 작은 f(n-1), f(n-2), ... 로 표현 가능한가?
```

재귀 트리를 그려서 같은 인자로 호출되는 지점이 보이면, 그 지점이 메모이제이션의 대상이다.

```java
// 피보나치: 재귀 → 메모이제이션 → 바텀업
Map<Integer, Long> memo = new HashMap<>();

long fib(int n) {
    if (n <= 1) return n;
    if (memo.containsKey(n)) return memo.get(n);
    long result = fib(n - 1) + fib(n - 2);
    memo.put(n, result);
    return result;
}
```

## 탑다운 vs 바텀업

| 방식 | 장점 | 단점 |
|------|------|------|
| 탑다운(메모이제이션) | 직관적, 필요한 상태만 계산 | 재귀 오버헤드, 스택 오버플로우 위험 |
| 바텀업(타뷸레이션) | 반복문, 캐시 친화적 | 모든 상태를 사전에 채워야 함 |

대부분의 알고리즘 문제에서 바텀업이 더 빠르다. 재귀 함수 호출 비용과 콜 스택 한계가 없기 때문이다.

## 상태 공간 줄이기

상태 정의에 따라 메모리 사용량이 크게 달라진다. 배낭 문제(Knapsack)를 예로 들면:

```java
// 2D DP: O(n * W) 공간
int[][] dp = new int[n + 1][W + 1];
for (int i = 1; i <= n; i++) {
    for (int w = 0; w <= W; w++) {
        dp[i][w] = dp[i - 1][w];
        if (w >= weight[i]) {
            dp[i][w] = Math.max(dp[i][w], dp[i - 1][w - weight[i]] + value[i]);
        }
    }
}

// 1D DP: O(W) 공간 (이전 행에만 의존하는 구조 활용)
int[] dp = new int[W + 1];
for (int i = 0; i < n; i++) {
    for (int w = W; w >= weight[i]; w--) {
        dp[w] = Math.max(dp[w], dp[w - weight[i]] + value[i]);
    }
}
```

이전 행에만 의존하는 점화식이라면 1D 배열로 공간을 절반 이하로 줄일 수 있다.

## 구간 DP와 2D DP

문제의 복잡도가 높아지면 상태 차원도 늘어난다.

```
구간 DP: dp[l][r] = 구간 [l, r]에 대한 최적값
2D DP:   dp[i][j] = i번째 원소와 j번째 원소까지의 최적값
```

행렬 체인 곱셈이나 최장 공통 부분 수열(LCS) 같은 문제가 대표적이다.

```java
// LCS: dp[i][j] = s1[0..i]와 s2[0..j]의 LCS 길이
int[][] dp = new int[m + 1][n + 1];
for (int i = 1; i <= m; i++) {
    for (int j = 1; j <= n; j++) {
        if (s1.charAt(i - 1) == s2.charAt(j - 1)) {
            dp[i][j] = dp[i - 1][j - 1] + 1;
        } else {
            dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
        }
    }
}
```

## DP 설계 체크리스트

```
1. 상태를 정의하라: dp[i]가 정확히 무엇을 의미하는가?
2. 점화식을 세워라: dp[i]를 dp[i-1], dp[i-2], ...로 표현할 수 있는가?
3. 기저 사례를 잡아라: dp[0], dp[1]은 무엇인가?
4. 계산 순서를 결정하라: 작은 문제가 먼저 계산되는가?
5. 공간을 최적화하라: 이전 행/열에만 의존하는가?
```

DP에서 가장 어려운 부분은 점화식 자체가 아니라 **상태를 어떻게 정의하느냐**다. 상태가 올바르게 정의되면 점화식은 자연스럽게 따라온다.
